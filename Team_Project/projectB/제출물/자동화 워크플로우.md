
## 1. 개요

-  단순히 한 기능을 구현한 것이 아니라 **스케줄 트리거 → RSS 수집 → 주제 필터링 → AI 요약 → 썸네일 생성 → Notion 저장 → 에러 처리**까지
	- 자동화 파이프라인 전체 흐름을 연결

| 항목       | 내용                                                                                         |
| -------- | ------------------------------------------------------------------------------------------ |
| 자동화 툴    | **n8n** (셀프호스트, Docker) — 내장 노드·UI 설정만 사용 (Code/Execute Command 노드 미사용)                    |
| 실행 방식    | **스케줄 트리거로 매일 자동 실행** (사람 개입 없음)                                                           |
| 실행 시간    | **매일 14:00 · 18:00** (Asia/Seoul)                                                          |
| 스케줄 식    | CRON `0 14,18 * * *`, 타임존 `Asia/Seoul`                                                     |
| 전체 노드 수  | **25개**                                                                                    |
| 인증 필요 노드 | `OpenAI Summary`(OpenAI), `Get Existing Dedup Keys`·`Notion Save`(Notion) — 그 외 노드는 인증 불필요 |

## 3. 단계별 흐름도

```text
[1] Schedule Trigger ─ 매일 14:00 / 18:00 (Asia/Seoul, CRON "0 14,18 * * *")
                         ▼
[2] Get Existing Dedup Keys (Notion · Get Many, Simplify OFF)
    ─ DB의 모든 페이지(=중복 방지 키)를 1회 조회 (이후 Filter Unsaved가 참조)
                         ▼
[3] Reduce To One (Limit 1) ─ 이후 RSS가 1회만 실행되도록 항목 1개로 축소
        │
        ├───────────────────────────────┐   (두 출처를 병렬 수집)
        ▼                                ▼
[4a] Herald RSS Read              [4b] BBC RSS Read
     biz.heraldcorp.com/rss/google/it    feeds.bbci.co.uk/news/technology/rss.xml
        ▼                                ▼
[5a] Herald Keyword Filter        [5b] BBC Keyword Filter
     제목에 \bAI\b 또는 인공지능 포함        (동일 규칙)
     + 필수필드(title/link/pubDate/content) 검사
        ▼                                ▼
[6a] Herald Limit (20)            [6b] BBC Limit (20)
        │                                │
        └────────────────┬───────────────┘
                         ▼
[7] Merge (append) ─ 두 출처 후보 합산
                         ▼
[8] Add Source & AI Score (Set) ─ source(헤럴드/BBC) · ai_score(AI키워드 1/0) · pub_date_str(ISO)
                         ▼
[9] Sort ─ ai_score ↓, 그다음 pubDate ↓ (AI 우선 + 최신순)
                         ▼
[10] Generate URL Hash (Crypto SHA-256) ─ 후보 전체에 url_hash 부여
                         ▼
[11] Build Dedup Key (Set) ─ 후보 전체에 dedup_key = url_hash|guid 부여
                         ▼
[12] Filter Unsaved (Filter) ─ [2]에서 조회한 저장 키에 이미 있는 후보는 제거
     = 미저장 기사만 통과 (모두 저장됐으면 0건 → 저장 안 함)
                         ▼
[13] Select 1 Unsaved (Limit 1) ─ 미저장 후보 중 1위(AI 우선·최신순) 1건 선택
                         ▼
[14] OpenAI Summary (gpt-4.1-nano) ─ 3줄 요약 + 제목 한국어 번역(title_ko) 생성  ※재시도 2회
                         ▼
[15] Parse AI JSON (Set) ─ is_target · title_ko · summary_lines · image_prompt 추출
                         ▼
[16] Is AI/IT Target? (If · is_target === true)  ※필터가 이미 AI만 통과시켜 항상 true
        ├─ false → [Skip - Not Target] (NoOp, 로그) ═══▶ 종료
        │
        └─ true  ▼
[17] Pollinations Image Generate (HTTP GET · 무료) ─ image_prompt + 기사별 seed로 썸네일 생성  ※재시도 2회
                         ▼
[18] Normalize Fields (Set) ─ notion_title(=title_ko) · summary · url · date · source · dedup_key · thumb_url
                         ▼
[19] Notion Save (HTTP · POST /v1/pages) ─ 제목·요약문·원문링크·발행일시·출처·중복방지키·썸네일(URL) 저장  ※재시도 2회
                         ▼
                       ✅ 완료

[E1] Error Trigger ───▶ [E2] Error Notify (placeholder NoOp)
     └ 워크플로우 레벨에서 미처리 오류 발생 시 실행 로그에 기록 (필요 시 Slack/Email 노드로 교체)
```

## 4. 노드별 역할 (연결 구조)

| # | 노드 | n8n 타입 | 역할 |
| --- | --- | --- | --- |

| 2 | Get Existing Dedup Keys | notion (getAll, Simplify OFF) | DB의 모든 저장 키를 1회 조회 (중복 비교 기준) |
| 3 | Reduce To One | limit(1) | RSS가 1회만 실행되도록 항목 축소 |

| 5a/5b | Herald / BBC Keyword Filter | filter | **주제 필터** — 제목에 AI/인공지능 포함 + 필수필드 검사 |
| 6a/6b | Herald / BBC Limit | limit | 출처별 최대 20건으로 제한 |
| 7 | Merge | merge (append) | 두 출처 후보 합산 |
| 8 | Add Source & AI Score | set | source·ai_score·pub_date_str 부여 |
| 9 | Sort | sort | ai_score↓ → pubDate↓ (AI 우선·최신순) |
| 10 | Generate URL Hash | crypto | 원문 URL SHA-256 해시 (후보 전체) |
| 11 | Build Dedup Key | set | `dedup_key = url_hash\|guid` (후보 전체) |
| 12 | Filter Unsaved | filter | 이미 저장된 후보 제거 (미저장만 통과) |
| 13 | Select 1 Unsaved | limit(1) | 미저장 후보 중 1위 1건 선택 |
| 14 | OpenAI Summary | langchain.openAi (gpt-4.1-nano) | **AI 요약(3줄) + 제목 한국어 번역** |
| 15 | Parse AI JSON | set | AI 출력 파싱(견고화) |
| 16 | Is AI/IT Target? | if | is_target=true만 진행 (필터가 이미 AI만 통과 → 사실상 항상 통과) |

## 5. 연결 구조의 핵심 설계

- **병렬 수집 → 병합**: 헤럴드·BBC RSS를 동시에 읽어 각각 필터링한 뒤 Merge로 합쳐, 두 출처를 하나의 후보 풀로 다룸
- **능동적 중복 방지**: 저장 직전 1건만 검사해 스킵하는 방식이 아니라, **시작에 저장 키 전체를 1회 조회 → 후보에서 미저장만 남김 → 그중 1위를 저장**..
	- 1위 기사가 이미 저장돼 있어도 **그다음 미저장 AI 기사를 능동적으로 찾아 저장**하므로, 매 실행이 "스킵"으로 끝나지 않음
- **비용 최소화**: AI 요약·이미지 호출은 **미저장 1건에 대해서만** 발생한다. 필수 필드·중복 검사를 AI 호출 전에 수행
- **에러 격리**: 워크플로우 레벨 미처리 오류는 Error Trigger 분기로 모아 로그에 남기고, 필요 시 Slack/Email 노드로 확장
