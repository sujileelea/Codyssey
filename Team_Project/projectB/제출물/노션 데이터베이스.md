
## 1. 데이터베이스 정보

| 항목            | 값                                                                        |
| ------------- | ------------------------------------------------------------------------ |
| DB 이름         | **AI/IT 뉴스 요약 자동화**                                                      |
| 저장 방식         | n8n `Notion Save` 노드 (HTTP `POST /v1/pages`) — 매 실행 시 미저장 AI 기사 1건 자동 저장 |

---

## 2. 속성(컬럼) 구성과 매핑

- 워크플로우가 저장하는 각 값이 **어느 Notion 속성에, 어떤 타입으로** 들어가는지 정리한 매핑
	- 과제 필수 항목인 **제목 · 요약문 · 원문 링크 · 발행 일시**가 모두 올바른 타입의 속성에 1:1로 매핑

| Notion 속성       | 타입        | 저장 값                                 |
| --------------- | --------- | ------------------------------------ |
| **제목**          | title     | AI가 번역한 한국어 제목(`title_ko`, 없으면 원제목)  |
| **요약문**         | rich_text | AI 3줄 요약 (`\n`으로 줄 결합)               |
| **원문 링크**       | url       | 기사 원문 URL                            |
| **발행 일시**       | date      | 기사 발행일 (YYYY-MM-DD)                  |
| **출처**          | select    | 헤럴드 / BBC                            |
| **중복 방지 키**     | rich_text | `url_hash\|guid` (SHA-256 해시 + GUID) |
| **썸네일(보너스 과제)** | files     | Pollinations 생성 이미지 URL (보너스)        |

## 4. 저장 콘텐츠 예시

> 아래는 저장 형식을 보여주는 템플렛

```text
제목      : 
요약문    :
  - 
  - 
  - 
원문 링크 :
발행 일시 : 
출처      : 
중복 방지 키 : 
썸네일    : 
```

- **요약문**은 AI(GPT-4.1 nano)가 원문 근거로만 생성한 **3줄 이내** 한국어 요약이며, 원문에 없는 내용은 추가하지 않는다(환각 방지)
- **제목**은 영어 기사일 경우 자연스러운 한국어로 번역해 저장(한국어 기사는 원제목 유지)
- **썸네일**은 보너스 과제로, 기사 주제에 맞춰 생성한 이미지 URL을 `썸네일`(Files) 속성에 외부 파일로 저장

## 5. 정확성 보장 포인트

- **속성 바인딩 안정화**: 네이티브 Notion 노드의 속성 매핑 이슈를 피하기 위해 저장을 **raw HTTP `POST /v1/pages`** 로 처리하고, 본문 JSON에 속성 매핑을 명시
	- 모든 속성이 올바른 위치에 적재
- **빈 값 방지**: 단일 기사 참조가 노드를 통과하며 끊기는 문제를 `.first()` 참조로 해결해, 제목·요약문 등 속성이 빈 값으로 저장되지 않도록 처리
- **발행일시 포맷**: RSS 발행일시를 Notion Date 형식(YYYY-MM-DD)으로 변환해 저장
