
## 0. 서비스 기획

- RSS로 최신 기술 뉴스를 자동 수집 + 생성형 AI로 **3줄 요약 + 제목 한국어 번역** + 썸네일과 함께 **Notion** 데이터베이스에 자동 저장하는 **노코드 자동화 파이프라인**
- n8n으로 스케줄 트리거·주제 필터링·AI 가공·중복 방지·에러 처리까지 전체 흐름을 연결

## 3. 주제 필터링 기준과 선택 이유

### 기준 (As-Built)
- **헤럴드·BBC 공통**: 기사 **제목에 `AI`(단어 경계 `\bAI\b`) 또는 `인공지능`이 포함**되면 통과
- 함께 **필수 필드 검사**(제목·원문 링크·발행일시·본문/설명)를 수행해 불완전한 기사를 제외
- 통과 기사에 `ai_score`(AI 키워드 포함=1)를 부여하고 **ai_score↓ → 발행일시↓** 정렬 후 **1건** 선택

### 선택 이유
- **단순·명확성**: "AI 뉴스인가"를 제목 한 줄로 판별 → 기준이 직관적이고 검수가 용이
- **본문 노이즈 차단**: 제목 기반이라 본문에 우연히 섞인 단어의 영향을 받지 않음
- **오탐 방지**: 단어 경계 `\bAI\b`를 써 `RAID`, `SAID` 같은 단어 속 "AI"를 필터링
- **언어 무관**: "AI"는 한국·영문 기사 모두 대문자 약어로 동일하게 쓰여 헤럴드·BBC에 같은 규칙 적용 가능(한국어 표기 `인공지능`도 보조 포함)

> 초기 설계는 출처별 다중 AI/IT 키워드(클라우드·사이버보안·IT 등) + 제외어 방식이었으나, 판별 기준을 명확히 하기 위해 "제목 AI" 단일 규칙으로 단순화

## 4. 에러 처리 정책과 선택 이유

- **핵심 원칙**: 재시도는 **최대 2회 상한**(무한 루프 금지), 요약 호출은 **기사 1건당 1회**, AI 호출 **전에** 필수 필드·중복 검사를 수행해 불필요한 비용 절감

| 상황 | 처리 | 선택 이유 |
| --- | --- | --- |
| RSS 미수집 / 필터 통과 기사 없음 | 로그 후 해당 회차 스킵 | 빈 데이터로 AI 호출·저장을 막아 비용·오류 방지 |
| 필수 필드 누락 | 필터에서 제외(저장 안 함) | 불완전 데이터가 DB에 쌓이는 것 방지 |
| 중복 기사 감지 | AI 호출 없이 스킵 | 중복 저장·불필요한 AI 비용 방지 |
| AI 요약 API 실패 / 응답 형식 오류 | **최대 2회 재시도** 후 로그 | 일시적 장애는 복구하되 무한 호출 비용 차단 |
| 이미지(썸네일) 생성 실패 | **최대 2회 재시도**, 최종 실패 시 하류 저장 미실행 | 썸네일 없는 결과물 저장 방지 |
| Notion 저장 실패 | **최대 2회 재시도** 후 로그 | 저장 누락을 확인 가능하게 |
| 워크플로우 레벨 미처리 오류 | Error Trigger → 로그(알림 노드 placeholder) | 실패를 한 곳에서 추적, 필요 시 Slack/Email로 확장 |

## 5. 중복 저장 방지 정책

- **중복 방지 키 = 원문 URL 해시(SHA-256) + RSS GUID** → `dedup_key = url_hash|guid` (GUID 없으면 `url_hash|`).
- **동작(As-Built)**: 실행 시작에 Notion DB의 **저장 키 전체를 1회 조회**(`Get Existing Dedup Keys`, Simplify OFF) → 후보 기사 전체에 dedup_key를 부여한 뒤 **`Filter Unsaved`로 이미 저장된 후보를 제거** → **`Select 1 Unsaved`로 미저장 중 1위(AI 우선·최신순) 1건**만 요약·저장.
- 이로써 1위가 이미 저장돼 있어도 **그다음 미저장 AI 기사를 능동적으로 찾아 저장**한다(스킵으로 끝나지 않음). 후보 전부가 저장된 경우에만 0건.
- AI/이미지 호출은 **미저장 1건에만** 발생 → 비용 절감.

## 6. 보너스 — 썸네일 이미지 자동 생성

- **설계**: GPT가 생성한 영어 `image_prompt`(기사 주제 기반)를 Pollinations.ai에 GET 요청해 썸네일을 생성(텍스트 없는 모던 에디토리얼 스타일)
- **저장**: 생성 이미지 URL을 Notion `썸네일`(Files & media) 속성에 외부 파일로 저장
- **무료 선택 이유**: OpenAI gpt-image-1과 Gemini imagen/flash-image는 모두 결제/유료 쿼터가 필요함을 라이브 테스트로 확인 → 비용 없이 운영하기 위해 무료·무가입 서비스 Pollinations 채택
- **기사별 다양성**: 같은 프롬프트면 같은 이미지가 나오는 문제를 막기 위해 URL에 **url_hash 기반 기사별 `seed`** 를 추가하고 `image_prompt`를 구체적 시각 묘사로 강화 → 기사마다 다른 썸네일

## 7. 보안 점검

- API 키·토큰(OpenAI, Notion 등)은 **n8n Credentials** 또는 **환경 변수(`.env`)** 에만 보관
- `.env`와 키 보관 파일(`KEY.rtf`)은 **`.gitignore`로 커밋 차단**한다(공개 저장소 노출 방지)
- 워크플로우 JSON 내 credential은 placeholder 참조로 두고, 실제 키는 n8n에서 연결
- 제출 문서·스크린샷에 인증 정보·토큰·시크릿이 보이지 않도록 마스킹을 확인

## 8. 재현(실행) 요약

1. n8n 실행 → OpenAI·Notion Credentials 등록
2. `app/workflow.json` Import → 노드 credential 연결(OpenAI 1개 + Notion 2개) → Sort 필드 확인
3. Execute로 수동 테스트 → Notion 저장(썸네일 포함) 확인 → 재실행으로 중복 방지 확인
4. **Activate**로 스케줄 자동 실행(매일 14:00·18:00) 시작
