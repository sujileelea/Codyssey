# 뉴스 요약 자동화 시스템 README / 기획서

## 1. 과제 핵심 요약

- RSS 피드에서 최신 기술 뉴스를 자동 수집한다.
- 사전에 정의한 주제 필터링 기준에 맞는 기사 1건을 선택한다.
- 생성형 AI를 사용해 기사 내용을 3줄 이내로 요약한다.
- 요약 결과를 노션 등 협업 도구의 데이터베이스에 자동 저장한다.
- 스케줄 트리거, 주제 필터링, AI 요약, 저장, 중복 방지, 오류 처리까지 포함한 자동화 파이프라인 전체를 설계하고 검증한다.
- 전체 과정은 사람의 수동 개입 없이 자동으로 완료되어야 한다.

## 5. 팀 작업 시퀀스 및 프로세스

### 5.1 전체 작업 순서

1. `a`가 과제 요구사항을 정리하고 사용할 자동화 툴, 저장 도구, AI 모델 후보를 확정한다.
2. `a`와 `d`가 노션 데이터베이스 속성 구조를 먼저 정의한다.
3. `b`가 기술 뉴스 RSS 피드를 선정하고 매일 실행되는 스케줄 트리거를 설정한다.
4. `b`가 RSS에서 제목, 링크, 본문, 발행일시를 추출한다.
5. `b`가 사전 정의된 키워드 또는 태그 기준으로 주제 필터링을 적용하고 기사 1건을 선택한다.
6. `c`가 선택된 기사 본문을 생성형 AI에 전달해 3줄 이내 요약문을 생성한다.
7. `e`가 요약문이 원문에 근거하는지, 3줄 제한을 지키는지, 필수 데이터가 누락되지 않았는지 품질게이트를 적용한다.
8. `d`가 요약 결과를 정규화하고 노션 DB 속성에 맞게 저장한다.
9. `e`가 오류 발생 시 재시도, 스킵, 로그, 알림 흐름을 검증한다.
10. `a`가 전체 워크플로우 스크린샷, 노션 저장 결과, README 내용을 통합해 최종 제출 상태를 점검한다.

### 5.2 워크플로우 구조

```text
[1] 스케줄 트리거
    - 매일 설정 시간 자동 실행
    - 타임존: Asia/Seoul
        ↓
[2] RSS 수집
    - 기술 뉴스 RSS 피드에서 최신 항목 수집
    - 제목, 링크, 본문, 발행일시 추출
        ↓
[3] 주제 필터링
    - 사전 정의한 키워드/태그 기준으로 기사 필터링
    - 조건을 만족하는 기사 1건 선택
        ↓
[4] AI 요약
    - 생성형 AI 모델 호출
    - 3줄 이내 요약 생성
    - 기사 1건당 요약 호출 1회 원칙 적용
        ↓
[5] 품질게이트
    - 요약문 길이, 원문 근거, 필수 필드, 중복 여부 확인
        ↓
[6] 노션 DB 저장
    - 제목, 요약문, 원문 링크, 발행일시, 중복 방지 키 저장
        ↓
[7] 예외 처리
    - 실패 시 최대 2회 재시도
    - 최종 실패 시 로그 기록, 스킵, 필요 시 알림
```

### 9.1 요약 프롬프트

```text
아래 뉴스 본문을 한국어로 3줄 이내로 요약해줘.
원문에 없는 내용은 추가하지 말고, 핵심 사실과 시사점만 포함해줘.

```

### 9.2 요약 규칙

- [ ] 3줄 이내로 요약한다.
- [ ] 원문에 없는 내용을 추가하지 않는다.
- [ ] 과장된 해석이나 환각을 방지한다.
- [ ] 기사 1건당 요약 호출은 1회만 수행한다.
- [ ] 실패로 인한 재시도는 최대 2회까지만 허용한다.

## 11. 안정성 및 오류 처리 정책

### 11.1 오류 상황별 처리

| 오류 상황 | 처리 정책 | 선택 이유 |
| --- | --- | --- |
| RSS에서 뉴스가 수집되지 않음 | 로그 기록 후 해당 실행 회차 스킵 | 빈 데이터로 AI 호출이나 저장을 하지 않기 위해 |
| 필터 조건을 만족하는 뉴스가 없음 | 로그 기록 후 스킵 | 주제와 무관한 기사를 저장하지 않기 위해 |
| AI 요약 API 호출 실패 | 최대 2회 재시도 후 실패 로그 기록 | 일시적 장애는 복구하되 무한 호출 비용을 방지하기 위해 |
| 노션 저장 실패 | 최대 2회 재시도 후 실패 로그 및 알림 | 저장 누락을 확인할 수 있도록 하기 위해 |
| 필수 필드 누락 | 저장하지 않고 오류 로그 기록 | 불완전한 데이터가 DB에 쌓이는 것을 방지하기 위해 |
| 중복 기사 감지 | 저장하지 않고 스킵 | 동일 기사 중복 저장을 방지하기 위해 |

### 11.2 재시도 정책

- [ ] 재시도는 최대 2회까지만 허용한다.
- [ ] 재시도 후에도 실패하면 로그를 남기고 해당 항목을 스킵한다.
- [ ] 무한 루프 또는 과도한 API 호출이 발생하지 않도록 설정한다.

### 11.3 중복 방지 정책

- 중복 방지 키: `GUID / 원문 링크 해시 / 원문 URL / 기타: ________`

- [ ] 각 단계별 역할과 연결 구조가 설명되어 있다.

- [ ] 주제 필터링 기준과 선택 이유가 작성되어 있다.
- [ ] 에러 처리 정책과 선택 이유가 작성되어 있다.
- [ ] 중복 저장 방지 키가 적용되어 있다.
- [ ] 요약 호출은 기사 1건당 1회 원칙을 따른다.
