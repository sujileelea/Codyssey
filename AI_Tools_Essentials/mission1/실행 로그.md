# 문서 목적

> 선택한 업무 과업 시나리오로 10턴 이상 실제 대화를 수행한 전문과, 문제 발생 지점(환각/문맥 끊김 등) 및 수정 결과 요약을 담는다.
> 대상 시스템은 **공고 분석 챗봇**(`app/`).

> ⚠️ 아래 대화는 시스템 동작과 형식을 보여주는 **예시 시나리오**다. 제출 시에는 실제 공고로 직접 수행한 대화 전문으로 교체하고, 원본 로그를 별도 파일로 보존한다.

---

# 1. 시나리오 설명

- 업무 과업: 공고 분석 챗봇으로 특정 공모전 공고를 분석하고 참여 가능 여부·일정·상금을 문답으로 확인
- 사용 모델 / 채널: `Qwen 3.6` / API(OpenAI 호환, Ollama 로컬)  ※실제 사용값으로 교체
- 수행 날짜(YYYY-MM-DD): 2026-__-__
- 포함된 변동 상황: (a) 7턴에서 **조건 변경**(대상 톤을 "사내 공유용"으로) (b) 9턴에서 **추가 정보 뒤늦게 제공**(참가 제약 조건)

# 3. 포함된 변동 상황

- 조건 변경 지점: **7턴** — 출력 대상/톤을 "사내 공유용·간결"로 변경
- 추가 정보 제공 지점: **9턴** — "4인 팀"이라는 제약 정보가 뒤늦게 제공됨 → 10턴 요약에 반영

# 4. 문맥 유지 검증

- 일관 유지된 요구사항: 1~6턴에서 확정된 제목·주관·일정·장소·대상·상금·마감 정보가 7·10턴 요약에서도 동일하게 유지됨. 정중·간결 톤 유지.
- 조건 변경 반영 결과: 7턴 톤 변경(사내 공유용)은 이후 요약(10턴)에도 계속 적용, 나머지 사실은 변경 없이 유지.
- 판정(Pass/Fail): **Pass** (확정 요구사항 일관 유지 + 변경분만 반영)
