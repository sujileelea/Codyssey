# 문서 목적

> 동일한 업무 과업 프롬프트(테스트 질문/케이스 포함)를 3종 이상의 LLM에 입력하고, 모델별 결과 비교표와 최종 선정 결론(근거 포함)을 정리한다.
> 대상 과업은 **공고 분석 챗봇**(`app/`)이며, 비교 모델은 Gemma 4 E4B / Llama 4 Scout / Qwen 3.6 이다.

> ⚠️ 점수표(4장)·결과 요약(5장)·재현성 날짜(7장)의 수치는 **실제 실행 후 본인이 측정한 값으로 채워야 한다.** 아래는 채점 형식과 기입 예시이며, 그대로 제출용 사실로 사용하지 말 것.

---

# 1. 비교 목적

공고 분석 챗봇은 ① 포스터+본문에서 사실 정보를 정확히 추출(Vision)하고, ② 추출된 컨텍스트에 근거해 환각 없이 답해야 한다. 세 모델 모두 멀티모달·한국어 처리가 가능하나 성능 편차가 있으므로, **동일 입력·동일 평가 축**으로 비교해 본 과업에 가장 적합한 모델을 선정한다.

# 2. 비교에 사용한 동일 입력

## 2-1. 업무 과업 프롬프트(공통 입력)

- 정보추출용 시스템 프롬프트: `시스템 설계 문서.md` 12-2 전문 사용
- 챗봇 응답용 시스템 프롬프트: `시스템 설계 문서.md` 12-1 전문 사용
- 동일한 테스트 공고 1건의 URL을 세 모델에 똑같이 입력 (분석 → 동일 질문 세트 문답)
- 테스트 공고 URL: `①여기에 실제 사용한 공고 URL 기입`

## 2-2. 테스트 질문 / 케이스

> 세 모델에 동일하게 던진 질문 세트(환각 검증 질문 포함).

1. 이 공고 제목과 주관기관은?
2. 행사 일정과 장소는? (원문 표기로)
3. 참여 대상/조건은? 비전공자도 가능한가?
4. 대상(1등) 상금 금액은?
5. 접수 마감일은 언제인가?
6. (함정) 원문에 없는 항목 질문 → "확인되지 않음" 처리하는가?
7. (모호) "이거 신청하면 돼?" → 되묻는가?

# 6. 최종 선정 결론

> 왜 이 모델이 공고 분석 과업에 최적인지 3줄 이상 근거로 설명. (아래는 작성 가이드 — 실제 점수로 확정)

1. 공고 분석 과업의 핵심은 **포스터 Vision 인식 + 사실 수치 정확성 + 환각 억제**이므로, 이 세 축(2·1·4) 점수가 가장 높은 모델을 채택한다.
2. (예시 논리) Qwen 3.6이 포스터 내 일정·금액 표를 가장 정확히 읽고 JSON 형식 준수율이 높다면, 형식 안정성까지 고려해 최적이다.
3. 다만 로컬 자원 제약/속도가 중요하면 경량 모델(Gemma)과의 트레이드오프를 명시한다. → 최종 채택: `②측정 후 모델명 기입` / 채택 사유: `③3줄 근거 기입`
