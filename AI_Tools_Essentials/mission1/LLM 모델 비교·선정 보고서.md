## 4-1. 평가 축별 채점 근거

> 각 축에서 모델별로 관찰한 구체 사례. (원문 전문: `archive/실행로그_원본_2026-06-24.md`)

- **1) 정확성** — 마감일(2026-06-28)·최우수상 250만원은 3모델 모두 정확. 단 Llama는 특별상 수여기관을 원문에 없는 "한국과학기술기획평가원장상"으로 생성(환각). 주관기관도 Gemma만 '대구디지털혁신진흥원'(주관)으로 정확히 구분, Llama·Qwen은 '대구광역시'(주최)로 표기.
- **2) 포스터 Vision 인식** — 상금표·상세일정은 본문에 없고 **포스터에만 존재**(Vision 필수). Gemma는 시상표 2개 부문 + 상세일정(서류평가 6/29~7/3, 발표평가 7월 중, 행안부 본선 9~11월)까지 모두 추출. Qwen은 시상표 2개 부문(최우수 250/우수 150/특별 100만원) 정확 추출하나 상세일정은 누락. Llama는 상세일정 누락 + 시상표를 1세트로 축약.
- **3) 한국어 자연스러움** — 3모델 모두 번역투 없이 자연스러운 존댓말. Gemma·Qwen은 항목형·소제목 구성이 정돈됨. Llama는 의미 전달엔 문제 없으나 "공고에는 3개의 상 종류와 상금이 명시되어 있습니다"처럼 다소 기계적.
- **4) 환각 억제** — 장소·주차(원문 미기재)는 3모델 모두 "확인되지 않습니다"+문의처(Pass). 마감 일수 계산(함정)도 3모델 모두 임의계산 회피. 그러나 모호질문("이거 신청하면 돼?")에 **Gemma만 자격조건을 제시하며 되물음(Pass)**, Llama·Qwen은 '신청 방법'으로 단정(Fail). 추가로 Llama는 상금 수여기관 환각.
- **5) 형식 준수** — max_tokens=8192에서 3모델 모두 11개 필드 JSON 파싱 성공. 단 Gemma는 `<thought>` 추론 토큰을 길게 출력해 기본값(2048)에선 JSON이 잘려 파싱 실패 → 출력 토큰 여유 필요(운영 주의).
- **6) 10턴 문맥 유지** — 3모델 모두 #10 요약본을 유지한 채 #11 팀정보(3명 a·b·c)를 #12에서 병합 출력(Pass). #11 조건 점검에서 Gemma·Qwen은 '대구 시민/대구 기업' 부문 요건까지 짚었고, Llama는 인원수만 확인.
- **7) 속도/안정성** — 분석(Vision): Llama 2.6초 ≪ Qwen 22.6초 ≪ Gemma 71.2초. 문답 평균: Llama 1.6초 / Qwen 5.6초 / Gemma 15.3초(최대 30초). 호출 오류는 3모델 모두 0건.
